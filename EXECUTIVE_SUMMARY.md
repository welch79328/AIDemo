# 業務行動成效評估系統 - 執行摘要

## 📌 專案目標

建立一個 **MVP（最小可行產品）系統**，幫助業務團隊：
1. **系統化記錄**客戶拜訪資料（基於「客戶健檢表 30 問」）
2. **量化評估**業務行動的成效與轉換率
3. **智能分類**客戶優先級（AA 客戶自動標記）
4. **追蹤導入**成功率（6 大 KPI 指標）

---

## 🎯 核心價值

### 對業務人員
- ⏱️ **節省時間**：資料整理時間減少 60%
- 🎯 **精準聚焦**：自動識別高價值 AA 客戶
- 📊 **成效可視**：即時查看個人業績與轉換率
- 📱 **隨時記錄**：移動端友善，拜訪現場即可輸入

### 對管理層
- 📈 **數據驅動**：完整的業務漏斗與轉換分析
- 👥 **團隊管理**：掌握每位業務的績效表現
- 🔍 **客戶洞察**：了解客戶分布與成熟度
- ✅ **導入追蹤**：監控簽約後的導入成功率

---

## 📊 基於研究的規劃架構

### 網路研究發現（2025 最佳實踐）

#### MVP 成功指標
- **用戶參與度**：DAU（每日活躍用戶）、留存率
- **財務指標**：CAC（客戶獲取成本）、轉換率
- **產品性能**：頁面載入 < 3 秒、錯誤率 < 1%

#### 業務 KPI 評估
- **領先指標**：客戶接觸次數、健檢完成率（可控制）
- **落後指標**：簽約數量、導入成功率（結果衡量）
- **平衡計分卡**：財務、客戶、流程、學習四維度

### Excel 分析洞察

#### 客戶健檢表架構（30 問）
```
一訪問題（A+B 類）
├── A. 公司基本資料（5 問必填）
│   └── 官網、LINE 使用、經營階段、集團背景
└── B. 物件狀態（5 問）
    └── 物件比例、案場規劃、分布地點、痛點

二訪問題（B+C+D 類）
├── B. 發展方向（4 問）
│   └── 擴大營運、設備需求、智能設備
├── C. 經營狀態（8 問必填）
│   └── 組織人力、帳務方式、大房東、租客類型
└── D. 數位化程度（3 問必填）
    └── 會計部門、作業系統、競品認知
```

#### AA 客戶判定條件（4 個自動標記）
1. ✅ 規劃擴大營運（新案場）
2. ✅ 代管需製作損益表給大房東
3. ✅ 包租 100-200 戶以上需製作差額發票
4. ✅ 有外籍租客

#### 導入成功 6 大 KPI
| 指標 | 成功標準 |
|-----|---------|
| 物件管理 | 上傳率 ≥ 50% |
| 合約管理 | 建立率 ≥ 50% |
| 帳務管理 | 定期編輯和發送 |
| 金流整合 | 成功串接並了解操作 |
| 自動通知 | 租客加入 Line@ |
| SOP 操作 | 7 天內有使用紀錄 |

## 🏗️ 系統架構設計

### 核心功能模組（MVP v1.0）

#### 1️⃣ 業務拜訪管理
- 建立客戶與拜訪記錄
- 追蹤拜訪狀態（待一訪 → 已一訪 → 待二訪 → 已二訪 → 已簽約）
- 待辦事項提醒

#### 2️⃣ 客戶健檢表單
- **一訪表單**：A 類（公司基本資料）+ B 類（物件狀態）
- **二訪表單**：B 類（發展方向）+ C 類（經營狀態）+ D 類（數位化）
- 智能表單驗證與草稿儲存

#### 3️⃣ 客戶分類與評分
- **AA 客戶自動標記**：符合 4 個條件之一
- **成熟度評分**（0-100 分）：
  - 規模指標（30%）：戶數、人數
  - 數位化程度（25%）：系統、會計部門
  - 成長潛力（25%）：擴大計畫、新案場
  - 痛點匹配度（20%）：系統可解決的痛點數

#### 4️⃣ 業務成效儀表板
**個人指標卡片：**
- 本月拜訪次數（一訪/二訪/追蹤）
- 簽約數量 & 轉換率
- AA 客戶獲取數

**轉換漏斗視覺化：**
```
接觸(45) → 一訪(28) → 二訪(15) → 簽約(5)
  100%      62%        54%        33%
```

**待辦事項清單：**
- 近期排程拜訪
- 待追蹤客戶

## 💻 技術方案

### 推薦技術堆疊

#### 前端（Web + Mobile PWA）
```
Next.js 14 + React + TypeScript
├── UI 組件：Shadcn/ui + Tailwind CSS
├── 表單管理：React Hook Form + Zod
├── 狀態管理：Zustand
└── 圖表：Recharts
```

#### 後端
```
Node.js + Express + Prisma ORM
└── 資料庫：PostgreSQL
```

#### 部署（MVP 低成本方案）
```
├── 前端：Vercel（免費）
├── 後端：Railway（$5-20/月）
└── 資料庫：Supabase Free Tier
總成本：$5-20/月
```

### 資料庫設計亮點

#### 核心資料表
1. **users** - 業務人員
2. **customers** - 客戶（含 AA 標記、成熟度評分）
3. **visits** - 拜訪記錄（JSON 儲存問卷答案）
4. **contracts** - 簽約記錄（含 6 大 KPI 追蹤）
5. **performance_metrics** - 業務績效（日/週/月彙總）

#### 彈性設計
- 使用 **JSON 欄位**儲存問卷答案（支援未來問題調整）
- **question_templates** 表可自訂問題
- **aa_customer_criteria** 表可設定 AA 判定條件
