# 🎯 AI 對談分析展示頁面使用指南

## 📍 訪問位置

打開瀏覽器訪問：**http://localhost:5173/ai-analysis**

或從導航欄點擊：**AI 對談分析**

---

## ✨ 功能展示

### 1. 對談分析
- 🔍 自動分析業務與客戶的對談內容
- 📋 匹配「客戶健檢表 30 問」中的問題
- 💯 顯示每個答案的信心度 (0-100%)
- 📝 提供對談證據引用

### 2. 客戶資訊提取
- 🏢 公司名稱
- 📊 物件數量
- 👥 人員數量
- 🏷️ 業務類型（包租/代管/混合）
- 💡 主要痛點分析

### 3. AA 客戶評估
- ⭐ 自動判定是否為 AA 客戶（高價值客戶）
- 📈 評分 (0-100 分)
- 🎯 信心度顯示
- 📝 詳細判定原因

---

## 🚀 快速開始

### 方式 1: 使用內建範例

頁面提供兩個範例對談：

1. **範例 1: AA 客戶** - 大型業者 (250間物件)
   - 點擊「載入範例 1」按鈕
   - 點擊「開始分析」

2. **範例 2: 小型業者** - 個人創業 (15間物件)
   - 點擊「載入範例 2」按鈕
   - 點擊「開始分析」

### 方式 2: 貼上對談記錄

1. 複製您的對談記錄
2. 貼到左側文字框
3. 點擊「開始分析」

範例格式：
```
你
 你們現在大概幾間啊？
業者
 一百多。
你
 那平常租客問題是誰在回？LINE嗎？
業者
 對，LINE為主。
```

### 方式 3: 上傳文件

1. 準備 `.txt` 格式的對談記錄
2. 拖曳文件到上傳區域，或點擊上傳
3. 點擊「開始分析」

## 📊 分析結果說明

### 匹配的問卷問題
- **問題編號**: 對應「客戶健檢表 30 問」的題號
- **問題內容**: 完整的問卷問題
- **答案**: AI 從對談中提取的答案
- **信心度**: 90%+ (高信心) | 70-89% (中等) | <70% (低信心)
- **證據**: 對談中支持此答案的片段

### AA 客戶判定標準
✅ **AA 客戶特徵**:
- 物件數量 100+ 間
- 有擴大營運計劃
- 完整組織架構
- 有外籍租客
- 需要製作差額發票 (包租 100-200戶以上)
- 大房東數量多 (代管)

## 💡 使用技巧

### 1. 對談記錄格式建議
- 清楚標示「你」和「業者」
- 保持原始對談格式
- 包含完整問答，不要截斷

### 2. 獲得更準確分析
- 提供更多對談內容（建議 10 輪以上問答）
- 包含具體數字（物件數、人數等）
- 提及明確需求和痛點

### 3. 實際應用
- 拜訪後立即分析對談
- 快速評估客戶價值
- 自動填寫問卷答案
- 識別關鍵商機

## 🔧 技術架構

### 後端
- **OpenAI API**: gpt-4o-mini 模型
- **FastAPI**: 提供 RESTful API
- **Pydantic**: 資料驗證

### 前端
- **Vue 3**: 響應式 UI
- **Element Plus**: UI 組件庫
- **Pinia**: 狀態管理

### API 端點
```
POST /api/v1/ai/analyze-conversation    # 對談分析
POST /api/v1/ai/extract-customer-info   # 客戶資訊提取
POST /api/v1/ai/assess-aa-customer      # AA 客戶評估
GET  /api/v1/ai/questionnaire           # 取得問卷結構
```

## 🎯 實際業務流程整合

### 建議工作流程

1. **拜訪前**: 準備問卷
2. **拜訪中**: 記錄對談（錄音或筆記）
3. **拜訪後**:
   - 整理對談記錄為文字
   - 上傳到 AI 分析頁面
   - 查看匹配的問卷答案
   - 確認 AA 客戶判定
4. **建立拜訪記錄**: 使用分析結果填寫問卷
5. **客戶分類**: 根據 AA 評估調整跟進策略

## 🚨 注意事項

1. **API Key 安全**
   - OpenAI API Key 已配置在 `.env` 文件
   - 請勿將 Key 提交到版本控制

2. **成本控制**
   - 使用 gpt-4o-mini 模型（每百萬 token $0.15）
   - 單次分析約消耗 1000-2000 tokens
   - 預估成本：每次分析 < $0.01

3. **隱私保護**
   - 對談記錄會發送到 OpenAI
   - 請確保遵守隱私政策
   - 生產環境建議加密傳輸
