# 業務行動成效評估系統 - MVP 專案規劃

## 📋 專案概述

### 專案目標
建立一個業務行動成效評估系統，幫助業務團隊追蹤和評估每次業務拜訪的成效，基於「客戶健檢表 30 問」的架構，提供數據驅動的決策支持。

### 核心價值主張
- **即時追蹤**：記錄每次業務拜訪的關鍵資訊
- **成效評估**：量化業務行動的轉換率和成功指標
- **智能分析**：基於歷史數據提供客戶分類和優先級建議
- **導入預測**：評估客戶導入系統的成功機率

---

## 🎯 基於網路研究的關鍵發現

### MVP 成功衡量指標（參考國際最佳實踐）
1. **用戶參與度指標**
   - Daily Active Users (DAU)：每日活躍業務人員數
   - 用戶留存率：持續使用系統的業務比例

2. **財務指標**
   - Customer Acquisition Cost (CAC)：獲取新客戶的成本
   - 轉換率：從接觸到簽約的轉換比例

3. **產品性能指標**
   - 頁面載入時間 < 3 秒
   - 系統錯誤率 < 1%

### 業務 KPI 評估最佳實踐（2025）
- **領先指標**：客戶接觸次數、健檢完成率
- **落後指標**：簽約數量、導入成功率
- **平衡計分卡方法**：結合財務、客戶、流程、學習成長四個維度

## 📊 基於客戶健檢表的系統架構分析

### 一、客戶健檢表 30 問架構解析

#### A. 公司/個人基本資料（一訪必問）
1. 公司官網/FB 網站
2. 是否使用 LINE 個人/LINE OA 管理租客
3. 公司/品牌名稱
4. 公司/個人經營階段（個人戶/準備成立/剛成立/物件量增加）
5. 公司老闆/集團背景（仲介/建設/家族/裝修/創業/旅館/兼職）
6. LINE 群組/唐三藏
7. 公司決策人員
8. 公司決策人員聯絡資訊

#### B. 物件狀態與發展方向（一訪+二訪）
**一訪：**
1. 物件比例（包租/代管）
2. 客戶的案場規劃（共生宅/套雅房/整層/透套/共享辦公室）
3. 公司分布地點（分布/總部）
4. 是否有經營社宅
5. 客戶目前的難題與痛點

**二訪：**
1. 是否有規劃擴大營運（AA 客戶標記）
2. 公司/決策者目標（戶數增加/人員異動/人力減少）
3. 客戶設備需求量（新案場/舊案場更新）
4. 是否有使用智能設備（電子鎖/攝影機/電表）

#### C. 經營狀態（二訪必問）
1. 公司組織人力數據（會計/業務/經理/總戶數/分工）
2. 金流：帳務方式（紙本/系統/外包會計）
3. 代管：大房東數量（AA 客戶：需製作損益表）
4. 包租：是否需製作差額發票（AA 客戶：100-200 戶以上）
5. 租客取向（高/中/低資產類型）
6. 是否有外籍租客（AA 客戶）
7. 是否加入公會（成熟度判斷）
8. 是否有合作對象（裝潢/清潔/水電）

#### D. 客戶數位化程度與決策影響點（二訪必問）
1. 是否有會計部門及人數
2. 公司作業方式（紙本/系統/協力系統）
3. 客戶目前的競品（飛豬/DDROOM/包管家）

## 🎯 導入成功 KPIs（基於 Excel 分析）

| 導入指標 | KPI 量化方式 | 成功標準 |
|---------|------------|---------|
| 物件管理 | 客戶是否已成功上傳 50% 以上管理物件 | ✅ 物件上傳率 ≥ 50% |
| 合約管理 | 上傳的物件中，是否已有 50% 以上物件建立合約 | ✅ 合約建立率 ≥ 50% |

## 🏗️ MVP 系統架構設計

### 第一階段：核心功能（MVP v1.0）

#### 1. 業務拜訪管理模組
**功能需求：**
- 建立客戶拜訪記錄
- 記錄一訪/二訪資料（基於 30 問架構）
- 拜訪狀態追蹤（待一訪/已一訪/待二訪/已二訪/已簽約/未成交）
- 待辦事項提醒（下次拜訪時間）

**資料結構：**
```
客戶拜訪記錄
├── 基本資訊
│   ├── 客戶名稱
│   ├── 聯絡人
│   ├── 聯絡方式
│   └── 拜訪日期
├── 一訪資料（A+B 類問題）
├── 二訪資料（B+C+D 類問題）
└── 狀態追蹤
    ├── 當前階段
    ├── 下次拜訪時間
    └── 備註
```

#### 2. 客戶健檢表單模組
**基於 30 問的數位化表單：**

**A 類：公司基本資料（5 個必填項）**
- 公司名稱、官網、LINE 使用狀況
- 經營階段（下拉選單）
- 集團背景（多選）

**B 類：物件與發展（5+4 項）**
- 一訪：物件比例、案場規劃、分布地點、痛點分析
- 二訪：擴大營運計畫、設備需求

**C 類：經營狀態（8 項必填）**
- 組織人力、帳務方式、大房東數量
- 租客類型、合作夥伴

**D 類：數位化程度（3 項必填）**
- 會計部門、作業系統、競品使用狀況

#### 3. 客戶分類與評分模組
**AA 客戶標記條件（自動判斷）：**
- ✅ 規劃擴大營運
- ✅ 代管需製作損益表給大房東
- ✅ 包租 100-200 戶以上需製作差額發票
- ✅ 有外籍租客

**客戶成熟度評分（0-100 分）：**
- 規模指標（30%）：管理戶數、公司人數
- 數位化程度（25%）：使用系統、會計部門
- 成長潛力（25%）：擴大營運計畫、新案場
- 痛點匹配度（20%）：系統可解決的痛點數量

#### 4. 業務成效儀表板
**關鍵指標展示：**

**個人業務指標：**
- 本月拜訪次數
- 一訪完成數 / 二訪完成數
- 簽約轉換率
- AA 客戶獲取數

**團隊整體指標：**
- 總拜訪次數
- 階段漏斗轉換率（接觸 → 一訪 → 二訪 → 簽約）
- 平均拜訪週期
- 導入成功率（基於 6 大 KPI）

**客戶分類統計：**
- 客戶經營階段分布
- 客戶規模分布
- 客戶數位化程度分布
- AA 客戶比例

## 📐 資料模型設計

### 核心資料表

#### 1. customers（客戶主表）
```sql
CREATE TABLE customers (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    company_name VARCHAR(255) NOT NULL,
    contact_person VARCHAR(100),
    contact_phone VARCHAR(20),
    contact_email VARCHAR(255),
    website VARCHAR(255),
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW(),
    created_by UUID REFERENCES users(id),

    -- 分類標記
    is_aa_customer BOOLEAN DEFAULT FALSE,
    customer_stage VARCHAR(50), -- 個人戶/準備成立/剛成立/物件量增加
    maturity_score INTEGER, -- 0-100

    -- 索引
    INDEX idx_company_name (company_name),
    INDEX idx_is_aa (is_aa_customer)
);
```

#### 2. visits（拜訪記錄表）
```sql
CREATE TABLE visits (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    customer_id UUID REFERENCES customers(id),
    visit_type VARCHAR(20), -- '一訪' | '二訪' | '追蹤'
    visit_date DATE NOT NULL,
    visit_status VARCHAR(20), -- '已排程' | '已完成' | '已取消'

    -- 一訪資料（JSON 格式儲存問卷答案）
    basic_info JSONB, -- A 類問題
    property_status JSONB, -- B 類問題（一訪部分）

    -- 二訪資料
    development_plan JSONB, -- B 類問題（二訪部分）
    business_status JSONB, -- C 類問題
    digitalization JSONB, -- D 類問題

    notes TEXT,
    next_action VARCHAR(255),
    next_visit_date DATE,

    created_at TIMESTAMP DEFAULT NOW(),
    created_by UUID REFERENCES users(id),

    INDEX idx_customer_id (customer_id),
    INDEX idx_visit_date (visit_date)
);
```

#### 3. contracts（簽約記錄表）
```sql
CREATE TABLE contracts (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    customer_id UUID REFERENCES customers(id),
    visit_id UUID REFERENCES visits(id),

    contract_date DATE NOT NULL,
    contract_type VARCHAR(50), -- 包租/代管
    property_count INTEGER,
    monthly_value DECIMAL(10, 2),

    -- 導入 KPI 追蹤
    kpi_property_upload_rate DECIMAL(5, 2), -- 物件上傳率
    kpi_contract_creation_rate DECIMAL(5, 2), -- 合約建立率
    kpi_billing_active BOOLEAN, -- 帳單已發送
    kpi_payment_integrated BOOLEAN, -- 金流串接
    kpi_notification_setup BOOLEAN, -- 自動通知
    kpi_sop_established BOOLEAN, -- SOP 建立

    onboarding_success BOOLEAN DEFAULT FALSE,
    onboarding_date DATE,

    created_at TIMESTAMP DEFAULT NOW(),

    INDEX idx_customer_id (customer_id),
    INDEX idx_contract_date (contract_date)
);
```

#### 4. users（業務人員表）
```sql
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(100) NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255),
    role VARCHAR(20), -- 'sales' | 'manager' | 'admin'
    team VARCHAR(50),

    created_at TIMESTAMP DEFAULT NOW(),
    last_login TIMESTAMP
);
```

#### 5. performance_metrics（業務績效表）
```sql
CREATE TABLE performance_metrics (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id),
    metric_date DATE NOT NULL,

    -- 活動指標
    total_visits INTEGER DEFAULT 0,
    first_visits INTEGER DEFAULT 0,
    second_visits INTEGER DEFAULT 0,

    -- 結果指標
    contracts_signed INTEGER DEFAULT 0,
    aa_customers_acquired INTEGER DEFAULT 0,

    -- 轉換率（自動計算）
    conversion_rate DECIMAL(5, 2),

    created_at TIMESTAMP DEFAULT NOW(),

    INDEX idx_user_date (user_id, metric_date),
    UNIQUE (user_id, metric_date)
);
```

## 🔒 安全性考量

### 資料安全
- ✅ 所有密碼使用 bcrypt 加密儲存
- ✅ HTTPS 強制加密傳輸
- ✅ JWT Token 有效期限管理
- ✅ SQL Injection 防護（使用 ORM）
- ✅ XSS 攻擊防護

- **業務人員**：只能查看/編輯自己的拜訪記錄

- **管理員**：系統完整權限

### 資料備份
- 每日自動備份資料庫
- 保留最近 30 天備份
- 異地備份機制

## 🎯 關鍵成功因素

### 1. 業務團隊採納
- **策略**：
  - 早期邀請 2-3 位業務參與測試，收集反饋
  - 設計簡單易用的介面，降低學習門檻
  - 展示明確的效益（節省時間、提高轉換率）

### 2. 資料品質
- **策略**：
  - 強制必填重要欄位
  - 提供下拉選單減少輸入錯誤
  - 定期資料品質檢查與清理

### 3. 持續優化
- **策略**：
  - 每兩週一次用戶訪談
  - 追蹤使用數據（Google Analytics）
  - 快速迭代改進

### 4. 與現有流程整合
- **策略**：
  - 不完全取代現有工具，而是補充
  - 提供資料匯入功能
  - 支援匯出給其他系統使用
